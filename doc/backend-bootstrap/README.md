# Backend Bootstrap

## Что реализовано
- Создан базовый Rust backend-проект с точкой входа `src/main.rs`.
- Зафиксирована модульная структура монолита в `src/modules`:
  - `api`
  - `config`
  - `rpc`
  - `storage`
  - `indexer`
  - `jobs`
  - `metrics`
  - `logging`
- Добавлен базовый HTTP endpoint `GET /health` для проверки жизнеспособности процесса.
- Добавлено JSON-логирование через `tracing`/`tracing-subscriber`.
- Подготовлены заготовки инфраструктуры: `docker-compose.yml`, `Dockerfile`, `migrations/0001_init.sql`.

## Зачем это нужно
- Формирует стартовый каркас модульного монолита без смешивания доменной логики.
- Даёт единый entrypoint для дальнейшей реализации требований ТЗ.
- Позволяет поэтапно добавлять бизнес-функции, не ломая структуру проекта.

## Ограничения текущего этапа
- HTTPS, Basic Auth, mTLS к RPC и полнофункциональный REST API ещё не реализованы.
- Полная схема БД по ТЗ пока не добавлена.
- Миграция пока является placeholder.
