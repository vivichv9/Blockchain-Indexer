\introduction % Структурный элемент: ВВЕДЕНИЕ

Прямой доступ к блокчейн-данным через узел Bitcoin Core удобен для
низкоуровневого взаимодействия, однако плохо подходит для регулярных
прикладных выборок, аналитики и построения внешних API. Узел ориентирован на
поддержание консенсуса и обслуживание RPC-вызовов, но не предоставляет
готовый слой предметных представлений, таких как исторические балансы адресов,
наборы UTXO, фильтрация транзакций по адресам и диапазонам времени, а также
жизненный цикл задач индексирования \cite{btc_core_rpc}.

В прикладных сценариях это приводит к высокой нагрузке на RPC-интерфейс,
сложности повторного расчета агрегатов и риску неконсистентности данных при
reorg. Для решения этой проблемы требуется отдельная система индексации
ончейн-данных, которая отделяет контур получения исходной информации от
контура ее хранения и предоставления клиентам.

В рамках настоящей работы рассматривается проект
\texttt{Bitcoin Blockchain Indexer}, разрабатываемый как модульный
stateless-монолит. Система получает данные из узла, сохраняет их в
PostgreSQL \cite{postgres_docs}, отдает через REST API и поддерживает
административное управление jobs, мониторинг узлов, выдачу данных из mempool и
инструменты сопровождения. Целью ВКР является адаптация и формализация этих
наработок в виде полноценной пояснительной записки по теме
«Система индексации ончейн данных».
